# Internal documentation

These are working documents — design rationales, audit reports, performance experiments, milestone summaries — not user-facing docs.

They're kept around for history and so future-me (or contributors) can see how decisions were reached, but they shouldn't clutter the repo root and they shouldn't be the first thing a new visitor sees.

User-facing documentation lives in [`../`](../) (the parent `docs/` directory) and the project [README](../../README.md).
